# 可视化大屏项目使用&运维手册

> 适用对象：项目使用人员、演示人员、部署与维护人员  
> 适用系统：当前成品（全球/全国/各省三屏），无登录系统

## 1. 快速上手

## 1.1 项目包含什么

1. 本项目有 3 个页面（大屏）：
   - 全球大屏（global-screen）
   - 全国大屏（china-screen）
   - 各省大屏（province-screen）
2. 三个页面支持互相跳转（页面顶部“切换全球/切换中国/切换各省”按钮）。
3. 当前成品**没有登录流程**，打开页面即可使用。

## 1.2 本地快速启动（开发/演示）

1. 打开终端，进入项目根目录：`health-dashboard`
2. 一键启动三屏：
   - `npm run dev:all`
3. 启动成功后访问：
   - 全球：`http://localhost:5172/`
   - 全国：`http://localhost:5173/`
   - 各省：`http://localhost:5174/`

## 1.3 线上访问（Vercel）

1. 访问项目域名根路径（`https://你的域名`），会自动跳转默认入口页。
2. 也可以直接访问子路径：
   - `/global-screen/`
   - `/china-screen/`
   - `/province-screen/`
3. 旧路径兼容跳转已配置（`/bigbig_screen`、`/two_bigscreen`、`/screen`）。

## 1.4 快速上手建议（未进行优化/调整）

- 可以补一张“入口导航图”（三屏入口 + 主要按钮位置），帮助首次使用者更快上手。
- 可以补“演示模式默认流程”（先看全球，再全国，再各省），降低现场讲解难度。

## 2. 功能操作说明

## 2.1 通用操作（所有页面）

1. **页面切换按钮**
   - 操作：点击顶部切换按钮
   - 效果：跳转到对应页面（本地端口或线上子路径）
2. **鼠标悬停图表**
   - 操作：移动到图表数据点/柱/区域
   - 效果：显示 tooltip 详细数值
3. **窗口大小变化**
   - 操作：改变浏览器窗口尺寸
   - 效果：图表自动 resize，页面继续可用

## 2.2 全球大屏操作

1. **时间轴播放/暂停**
   - 操作：点击中部地图时间轴播放按钮
   - 效果：年份自动从 2000-2023 变化，地图热力随年份刷新
2. **时间轴拖动**
   - 操作：拖动时间轴滑块
   - 效果：立即切换到指定年份并重绘地图
3. **查看核心图表**
   - 左侧：WHO 区域缺口、全球趋势、风险因素占比
   - 中部：世界热力地图、LISA 空间分布
   - 右侧：堆叠指标、全球 vs 中国雷达、地区分布占比
4. **查看顶部指标卡**
   - 操作：页面加载后直接观察
   - 效果：显示全球关键指标并有动画过渡

## 2.3 全国大屏操作

1. **年份切换**
   - 操作：点击时间轴年份或播放/暂停
   - 效果：中心地图与左右图表联动更新
2. **地图查看**
   - 操作：悬停各省区域
   - 效果：查看省份老龄化率与相关医疗指标信息
3. **重点图表查看**
   - 左侧：散点图、双重心轨迹、堆叠柱
   - 右侧：供需双轴、Top10 条形、Top5 雷达
   - 底部：桑基图与滚动排行

## 2.4 各省大屏操作

1. **省份热力地图年份切换**
   - 操作：使用地图控制区的年份滑块
   - 效果：省份热力图按年份更新
2. **查看排行模块**
   - 操作：查看左右排行图
   - 效果：展示耦合度 Top10 与 OTE Top10
3. **查看趋势模块**
   - 操作：查看区域趋势与气泡图
   - 效果：展示区域指标变化与时间序列分布
4. **顶部时间**
   - 操作：页面打开后自动刷新
   - 效果：每秒更新时间显示

## 2.5 功能操作建议（未进行优化/调整）

## 3. 常见问题（FAQ）

以下问题均来自当前成品的真实运行场景与已出现日志，不虚构。

## 3.1 `npm run dev:all` 提示端口占用

**现象**  
启动时出现 `Port 5173 is already in use` / `Port 5174 is already in use`。

**原因**  
本机已有旧的 Vite 进程占用端口。

**处理步骤**
1. 重新执行 `npm run dev:all`（脚本会先尝试清理端口）。
2. 如果仍冲突，先关闭其他终端中占用 5172/5173/5174 的进程，再重试。

## 3.2 页面能打开但图表不显示/地图空白

**常见原因**
1. 数据文件路径加载失败；
2. 资源未按子路径正确访问；
3. 某个页面脚本未加载完成（各省页外链脚本模式）。

**处理步骤**
1. 确认是否通过标准路径访问（本地 5172/5173/5174，线上子路径）。
2. 打开浏览器 Network，检查数据文件请求是否 200。
3. 执行 `npm run vercel-build` 重新构建并验证 `public_out` 内容。

## 3.3 本地构建遇到 `EPERM unlink`（Windows）

**现象**  
`npm ci` 清理 `node_modules` 时提示 `EPERM`。

**原因**  
Windows 下原生二进制文件（rolldown/rollup/esbuild）被占用。

**处理步骤**
1. 关闭所有 dev server 后重试；
2. 直接使用根命令 `npm run vercel-build`（脚本内已对安装失败做回退处理）。

## 3.4 Vercel 构建失败（`npm run vercel-build exited with 1`）

**处理步骤**
1. 先看 Build Logs 定位失败发生在哪个子应用；
2. 确认仓库已包含最新 `scripts/vercel-build.mjs`；
3. 重新触发 Redeploy。

## 3.5 常见问题建议（未进行优化/调整）

- 可以增加一份“发布前检查清单”并固定到仓库（端口、构建、子路径、核心数据请求）。
- 可以补充错误码对照表（例如端口冲突、路径404、构建失败）提高排查效率。

## 4. 运维操作手册

## 4.1 部署环境

1. 部署平台：Vercel
2. 项目类型：静态前端多应用
3. 构建入口：根目录脚本

## 4.2 Vercel 配置（当前标准）

1. Root Directory：`./`
2. Build Command：`npm run vercel-build`
3. Output Directory：`public_out`

## 4.3 发布流程（标准）

1. 本地确认：
   - `npm run dev:all` 可启动三屏
   - `npm run vercel-build` 成功
2. 提交代码并推送：
   - `git add -A`
   - `git commit -m "chore: ..."`
   - `git push origin HEAD`
3. Vercel 自动或手动 Redeploy
4. 发布后验证：
   - 根路径可访问
   - 三个子路径可访问且刷新不 404
   - 核心地图/图表可显示

## 4.4 启停流程（本地）

1. 启动：
   - `npm run dev:all`
2. 停止：
   - 在终端按 `Ctrl + C`
3. 单屏启动（按需）：
   - `npm run dev:global`
   - `npm run dev:china`
   - `npm run dev:province`

## 4.5 数据备份/恢复（当前实现可执行方式）

当前项目为静态文件数据源，备份恢复以文件级操作为主。

1. 需要重点备份的文件：
   - 根目录 CSV：`health_datas.csv` 等
   - `apps/china-screen/public/data/*`
   - `apps/province-screen/public/data/*`
   - `apps/global-screen/public/world.geo.json`
2. 恢复方式：
   - 将备份文件覆盖回原路径
   - 重新执行 `npm run vercel-build`
   - 验证页面数据是否恢复正常

## 4.6 日志查看（当前实现）

1. 本地日志：
   - 直接查看运行终端输出（`dev:all` 或单屏 dev）
2. 构建日志：
   - 查看 `npm run vercel-build` 控制台输出
3. 线上日志：
   - 在 Vercel Deployment 的 Build Logs 查看失败阶段与错误信息

## 4.7 运维建议（未进行优化/调整）

- 可增加“每日自动构建巡检”任务，提前发现依赖或路径问题。
- 可把发布验证步骤脚本化（如健康检查脚本），减少人工漏检。

## 8. 操作路径详单（按场景展开）

1. 打开终端进入根目录。
2. 执行 `npm run dev:all`。

   - `http://localhost:5172/`

5. 在全国页测试年份播放与手动年份选择。

1. 打开根域名，确认默认入口可访问。
2. 直接访问并刷新：
   - `/global-screen/`
   - `/china-screen/`

## 9. 问题处理作业卡（Runbook）

## 9.1 作业卡 A：端口冲突

- **触发条件**：出现 `Port 517x is already in use`
- **标准动作**：
  1. 停止当前 dev 进程
  2. 重新执行 `npm run dev:all`
  3. 仍失败时关闭其他占用端口进程后重试
- **完成判定**：三端口均显示 `Local` 地址

## 9.2 作业卡 B：图表空白/地图空白

- **触发条件**：页面框架显示但核心图表为空
- **标准动作**：
  1. 检查访问路径是否正确（子路径或本地端口）
  2. 检查浏览器 Network 关键数据请求是否 200
  3. 执行 `npm run vercel-build` 并复核产物
- **完成判定**：至少一个关键地图和两类图表恢复显示

## 9.3 作业卡 C：Vercel 构建失败

- **触发条件**：`Command "npm run vercel-build" exited with 1`
- **标准动作**：
  1. 在 Build Logs 定位失败子应用
  2. 本地执行同命令复现并排查
  3. 修复后推送并 Redeploy
- **完成判定**：构建成功且上线检查清单通过

## 9.4 作业卡 D：Windows `EPERM unlink`

- **触发条件**：`npm ci` 期间出现 `EPERM unlink`
- **标准动作**：
  1. 停止正在运行的 dev 进程
  2. 重新执行 `npm run vercel-build`
  3. 若仍失败，重启终端后再次构建
- **完成判定**：构建命令完整执行结束且产物目录齐全

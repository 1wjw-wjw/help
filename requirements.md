# 需求文档

## 介绍

本项目为三屏医疗健康可视化大屏系统，包含全球、全国、各省三个页面，面向数据展示与研判场景。系统已完成前端可视化成品实现，支持本地三端口联调与 Vercel 子路径部署访问。

## 术语表

- **三屏系统**: 由全球大屏、全国大屏、各省大屏组成的整体应用
- **全球大屏**: `apps/global-screen`，展示全球健康资源与韧性相关可视化
- **全国大屏**: `apps/china-screen`，展示中国老龄化背景下资源错配可视化
- **各省大屏**: `apps/province-screen`，展示省级耦合度/OTE/区域指标可视化
- **时间轴联动**: 通过年份滑块或自动播放驱动地图和图表更新
- **子路径部署**: 生产环境以子目录路径访问各屏页面
- **静态数据源**: CSV/XLSX/GeoJSON 文件，前端按需加载与解析

## 需求

### 需求 1: 三屏页面与路由切换

**用户故事:** 作为演示用户，我希望在全球、全国、各省三屏之间切换，以便查看不同层级的数据视角。

#### 验收标准

1. 系统应提供三个可访问页面：全球、全国、各省
2. 每个页面应提供跨屏切换按钮（按页面现有按钮集合）
3. 在开发环境下，切换应跳转到对应端口（5172/5173/5174）
4. 在生产环境下，切换应跳转到对应子路径（`/bigbig_screen/`、`/two_bigscreen/`、`/screen/`）
5. 根路径访问应可重定向到默认入口页面

### 需求 2: 全球大屏整体展示

**用户故事:** 作为观众，我希望在一个页面中同时查看全球关键指标、地图和多类图表，以便快速理解全球健康资源分布与变化。

#### 验收标准

1. 页面应显示顶部 KPI 指标卡，包含预期寿命、卫生支出占比、5岁以下死亡率、DPT 接种率
2. 页面左侧应显示 WHO 区域缺口柱图、全球趋势折线图、健康风险环图
3. 页面中部应显示世界热力地图与时间轴控件
4. 页面中部应显示 LISA 聚类世界分布图
5. 页面右侧应显示堆叠条形图、雷达图、地区分布饼图
6. 页面应显示实时时间信息

### 需求 3: 全球大屏时间轴交互

**用户故事:** 作为分析用户，我希望按年份查看全球地图变化，并支持自动播放，以便观察 2000-2023 的时序变化。

#### 验收标准

1. 世界热力图应提供年份滑块（2000-2023）
2. 点击播放按钮时，应按固定节奏自动递增年份
3. 点击暂停按钮时，应停止自动播放并保留当前年份
4. 手动拖动年份时，应立即重绘世界热力图
5. 地图 tooltip 应显示国家名、D 值与预期寿命

### 需求 4: 全球大屏数据加载与容错

**用户故事:** 作为运维人员，我希望页面在数据源不可用时仍可展示基础内容，以保证演示稳定性。

#### 验收标准

1. 全球大屏应从静态 CSV 文件加载核心数据集
2. 当主文件候选路径失败时，应继续尝试其他候选路径
3. 世界地图应优先加载本地 `world.geo.json`，失败时可回退远程 geojson
4. 当部分聚合数据不可用时，应使用内置 fallback 数据结构完成图表渲染
5. 数据加载完成后应触发图表初始化与首帧渲染

### 需求 5: 全国大屏整体展示

**用户故事:** 作为分析用户，我希望查看中国老龄化与医疗资源错配关系的多维联动图表，以便进行省级比较和时序观察。

#### 验收标准

1. 页面应显示顶部四项关键指标卡并带数字动画
2. 中心区域应显示中国分级设色地图
3. 左侧应显示散点图、轨迹折线图、堆叠柱图
4. 右侧应显示双Y曲线图、Top10 条形图、Top5 雷达图
5. 底部应显示桑基图与滚动排行
6. 各图表应随当前年份联动更新

### 需求 6: 全国大屏时间联动与轮播

**用户故事:** 作为演示用户，我希望通过年份控制同步观察全国多图变化，以便展示年度趋势。

#### 验收标准

1. 页面应提供年份选择控件与播放/暂停按钮
2. 自动播放开启时应按固定间隔循环年份序列
3. 切换年份时，地图与相关图表应同步刷新
4. resize 后图表应保持可见并重排
5. 页面卸载时应清理定时器与图表实例

### 需求 7: 各省大屏整体展示

**用户故事:** 作为观众，我希望在各省页面查看耦合度、OTE与区域指标，以便理解省级层面的差异。

#### 验收标准

1. 页面应展示省份耦合度 Top10 排行模块
2. 页面应展示水质量雷达图模块
3. 页面应展示区域健康指标趋势折线模块
4. 页面应展示耦合度时间序列气泡图模块
5. 页面应展示省份热力地图模块及年份控制
6. 页面应展示 OTE 均值 Top10 排行模块

### 需求 8: 各省大屏数据与脚本加载

**用户故事:** 作为开发者，我希望各省页面在保留现有脚本架构下可稳定加载图表资源，以便保证历史实现可运行。

#### 验收标准

1. 各省页面应按现有机制顺序加载 ECharts/jQuery/XLSX/业务脚本
2. 页面应从 `public/data` 下读取 `map.xlsx`、`qipao.xlsx`、`雷达.xlsx`、`top-D.xlsx`、`OTE排名.xlsx`、`two.xlsx`
3. 地图数据读取失败时应尝试 `map.csv` 回退
4. 数据异常时模块应可降级为空图而不阻断页面其他模块
5. 页面应保留顶部实时时钟刷新

### 需求 9: 可视化渲染与交互一致性

**用户故事:** 作为用户，我希望所有图表在交互、提示与视觉反馈上保持一致，以便提升理解效率。

#### 验收标准

1. 所有图表应支持 tooltip 信息展示
2. 地图图表应支持视觉映射（`visualMap`）色阶表达
3. 图表容器尺寸变化时应执行 resize
4. 图表渲染失败不应导致整页崩溃
5. 页面应保持既有深色科技风视觉主题

### 需求 10: 页面加载与动画行为

**用户故事:** 作为观众，我希望页面首屏加载过程平滑，避免空白或突兀显示。

#### 验收标准

1. 页面应在数据与图表准备后进入首屏显示状态
2. 指标卡应按现有动画曲线更新显示值
3. 地图和图表应在容器可用后初始化
4. 页面切换后应加载目标页并显示对应模块
5. 页面卸载应释放已创建图表实例

### 需求 11: 数据需求与格式约束

**用户故事:** 作为维护人员，我希望明确系统依赖的数据类型与字段约束，以便保障运行一致性。

#### 验收标准

1. 系统数据源应包含 CSV、XLSX、GeoJSON 三类格式
2. 全球核心数据应包含国家、年份、U1/U2/U3、D 值等字段（按现有解析）
3. 全国核心数据应包含省份、年份及医疗资源/居民健康相关字段
4. 各省数据应按现有脚本规则从工作表中解析省份、年份与指标值
5. 非法数值应按现有逻辑做容错处理（置 0 或跳过）

### 需求 12: 非功能要求（性能与兼容）

**用户故事:** 作为使用者，我希望系统在主流桌面环境中稳定运行，并具备可部署能力。

#### 验收标准

1. 项目应基于 Vue3 + Vite 多应用结构运行
2. 开发环境应支持 `npm run dev:all` 一键启动三屏
3. 构建环境应支持 `npm run vercel-build` 输出统一产物目录
4. 部署环境应支持 Vercel 子路径访问与刷新路由
5. 系统应不依赖后端实时接口即可完成主要展示

### 需求 13: 非功能要求（安全与边界）

**用户故事:** 作为项目负责人，我希望明确系统边界能力，避免误解为权限平台或业务系统。

#### 验收标准

1. 系统不包含登录与权限鉴权模块
2. 系统不包含用户账户与会话管理
3. 系统不包含后端写入能力
4. 系统主要执行只读数据加载与前端可视化渲染
5. 系统部署后依赖静态资源可访问性

### 需求 14: 部署与运行规范

**用户故事:** 作为部署人员，我希望按照固定配置完成上线，以便快速稳定发布。

#### 验收标准

1. 根目录应提供统一构建命令 `npm run vercel-build`
2. 构建产物应输出到 `public_out`，包含三屏子目录与根入口文件
3. Vercel 配置应使用 Root Directory=`./`
4. Vercel 配置应使用 Build Command=`npm run vercel-build`
5. Vercel 配置应使用 Output Directory=`public_out`

## 附录: 当前实现范围声明

- 本文档仅描述当前项目已实现成品的功能与行为。
- 本文档不引入新增功能、不调整现有 UI/交互、不修改业务逻辑。

## 未进行的优化/调整建议

### 1. 功能层建议
- **地图联动增强**: 当前三屏独立展示，可补充跨屏共享年份或筛选状态（保持现有页面逻辑不变前提下做可选联动）。
- **交互提示完善**: 对空图、降级数据、加载失败状态增加统一可视提示文案。
- **参数化切换入口**: 将跨屏 URL 覆盖参数集中管理，减少环境切换时人工配置成本。

### 2. 数据层建议
- **数据模式校验**: 在前端解析前增加字段完整性检查与版本号校验，降低脏数据导致的静默异常。
- **缓存策略优化**: 对较大 CSV/XLSX 增加缓存与过期策略，减少重复解析开销。
- **数据预处理**: 将高频聚合逻辑前移到构建阶段，减轻运行时计算压力。

### 3. 可视化与体验建议
- **色标一致性规范**: 对多图 `visualMap` 和调色板建立统一规范，提升跨页面认知一致性。
- **动画节流策略**: 对高频重绘图表增加更细粒度节流/防抖，避免低端设备掉帧。
- **可读性优化**: 针对密集标签/图例场景增加自适应显示策略（折叠、抽样、滚动图例阈值）。

### 4. 性能建议
- **按需加载细化**: 对非首屏图表与大体量依赖继续拆分，降低首屏包体积。
- **渲染器策略评估**: 按图表类型评估 Canvas/SVG 最优组合，平衡清晰度与性能。
- **运行监控埋点**: 增加加载耗时、渲染耗时、错误率等指标采集，形成可观测基线。

### 5. 工程化建议
- **脚本日志标准化**: 为 `dev-all` 与 `vercel-build` 增加结构化日志前缀，提升排障效率。
- **质量门禁完善**: 补充格式检查、lint、构建检查的统一入口命令。
- **环境配置模板化**: 提供 `.env.example` 与部署变量说明，降低协作成本。

### 6. 测试与验收建议
- **关键流程自动化验收**: 增加三屏启动、跨屏跳转、核心图表渲染的自动化冒烟检查。
- **回归清单固化**: 将“白屏/404/资源加载/地图展示/图表空白”固化为发布前 checklist。
- **截图对比基线**: 建立关键页面视觉快照，降低样式与图表回归风险。

### 7. 部署与运维建议
- **多环境配置分层**: 区分开发、预发、生产构建变量，降低环境耦合。
- **静态资源策略**: 明确缓存头与失效策略，优化二次访问速度。
- **故障回滚预案**: 建立版本标记与一键回滚流程，缩短线上异常恢复时间。
